# Skill FitMangas — Sections & structure Reel

Source : guide LMDM montage Claude Code + HyperFrames, adapté FitMangas.

## Objectif

Découper la **transcription réelle** (post-dérush) en sections logiques.  
Ne pas inventer de sections basées seulement sur le brief écrit.

## Différence FitMangas vs LMDM

- LMDM : souvent **script lu** au prompteur.  
- FitMangas : **parole naturelle** — sections = ce qui a **vraiment** été dit (Whisper), pas le brief.

## Structure type (adapter au rush)

1. Accroche / hook  
2. Principe / insight  
3. Preuve ou explication  
4. Étape pratique (si dite)  
5. Bénéfice  
6. CTA / invitation (si dite)

Une section = **une idée claire entendue**. Le brief sert l’intention, pas à forcer des phrases non dites.

## Layout par défaut (LMDM)

- **Écran scindé** : motion / visuel **en haut**, visage recadré **en bas**  
- **Plein écran** : seulement si la section le mérite (visage peut disparaître derrière le visuel)  
- Respecter zones mortes IG (skill 07)

## Lien motion

Chaque section = terrain pour **une** idée visuelle (skill 03) — pas un feu d’artifice.
